## 1. One-Line Summary

MatryoshkaKV 通过可训练正交投影压缩 KV 的 hidden dimension，在尽量保留语义的同时显著减小 KV cache。

## 2. Why It Matters

很多 KV cache 方法主要在 token 维度做保留或淘汰，而 MatryoshkaKV 直接切到 `hidden dimension` 压缩。  
这条路线很重要，因为它补上了 repo 里和 eviction、quantization、merging 都不同的一支。

## 3. Main Idea

- 用 trainable orthogonal projection 将 KV 映射到更低维空间。
- 核心压缩轴不是 token 数量，而是每个 token 的 hidden/channel 表达维度。
- 支持分层或嵌套式压缩粒度，便于在不同预算下调节压缩率。
- 目标是在压缩率和表示保真之间建立更平滑的权衡。

## 4. Relation to Existing Work

- 相比 H2O、SnapKV、CAKE，它不主要回答“留哪些 token”，而是“每个 token 的 KV 能压到多小”。
- 相比 KIVI、KVQuant，它不是降 bit-width，而是降表示维度。
- 相比 Palu，它同属 hidden-dimension compression，但更强调可训练投影结构。

## 5. Evidence

- 论文主张在较高压缩率下仍能保持较好的下游质量。
- 核心收益是 `KV cache memory↓`，同时尽量控制 `quality drop↓`。

## 6. My Take

- 这篇论文值得收录，因为 `hidden-dimension KV compression` 是非常清晰的一条子路线。
- 它很适合和 Palu、KVTuner、KVQuant 放在一起对比不同压缩轴。
