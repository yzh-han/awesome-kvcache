## 1. One-Line Summary

ShadowKV 用低秩 key shadow 和 value offload，把长上下文 decode 的 KV 访问改成“先筛选、再重建”的高吞吐路径。

## 2. Why It Matters

它同时命中 `memory capacity` 和 `memory bandwidth` 两个瓶颈，属于很强的 serving/system 论文。

## 3. Main Idea

- 在 GPU 上保留低秩 key cache 作为 shadow。
- 将更重的 value cache 放到 host 侧。
- decode 时只重建少量稀疏 KV。
- 目标是扩大 batch 和支持更长上下文。

## 4. Relation to Existing Work

- 相比 SpeCache，它更偏 selection + reconstruction。
- 相比 H2O / SnapKV，它更偏 serving system 路线。
- 相比 HGCA，它不做双路 attention 融合。

## 5. Evidence

- 公开摘要强调 high-throughput long-context inference。
- 主要收益方向是 `memory↓`、`throughput↑`。

## 6. My Take

- 这是 2025 很值得收进主线的 offloading/system 代表作。
- 它补的是 repo 之前较弱的 `shadow / reconstruction` 子路线。
