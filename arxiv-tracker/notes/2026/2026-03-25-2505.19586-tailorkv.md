## 1. One-Line Summary

TailorKV 把 dominant-token loading 和 KV quantization 做成 layer-wise hybrid 优化，用不同层的不同角色来平衡 PCIe 开销和压缩误差。

## 2. Why It Matters

很多方法在 offload 和 compression 之间二选一，TailorKV 明确把两者做成互补组合。

## 3. Main Idea

- 分析哪些层更适合 selective loading，哪些层更适合量化。
- 对重要 token 做加载，对全量 token 做量化。
- 同时减轻 PCIe 瓶颈和压缩误差。

## 4. Relation to Existing Work

- 相比 SpeCache / NEO，它更强调 hybrid policy。
- 相比 KVTuner / XQuant，它不是纯量化路线。
- 相比 HGCA，它不让 CPU 直接参与 attention 计算。

## 5. Evidence

- ACL Anthology 摘要强调同时降低 memory overhead 和 loading latency。

## 6. My Take

- 这篇很适合放到 offloading 与 quantization 的交叉位置。
- 主分类放在 offloading 更合适，因为核心矛盾是 CPU-GPU 传输与加载。
