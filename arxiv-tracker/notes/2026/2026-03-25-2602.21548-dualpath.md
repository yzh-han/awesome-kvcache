## 1. One-Line Summary

这篇论文把 agentic LLM inference 的核心瓶颈重新定位为 **分离式架构下 KV cache 的存储带宽不对称**，并提出双路径 KV 加载与全局调度来提升吞吐。

## 2. Why It Matters

它关注的不是“如何压缩 KV”，而是“如何把已经存在的大 KV 更快搬到该去的地方”。这和当前 repo 里的 offloading / disaggregated serving 方向高度一致，而且问题定义很实在: prefill 侧存储 NIC 饱和、decode 侧 NIC 空闲。

## 3. Main Idea

- 指出常见 disaggregated inference 中，KV cache 从外部存储加载到 prefill engine 会让存储网络成为系统吞吐瓶颈。
- 提出 `dual-path KV loading`: 除了传统 `storage -> prefill` 路径，还增加 `storage -> decode -> prefill` 的路径。
- 第二条路径通过 RDMA 走计算网络，把 decode 侧空闲的网络能力利用起来，缓解 prefill 侧的 I/O 拥塞。
- 配合一个全局调度器，在 prefill 和 decode engine 之间动态平衡负载。
- 核心收益来自数据路径和调度协同，而不是模型或 attention 本身的变化。

## 4. Relation to Existing Work

- 相比 PagedAttention / vLLM，这不是单机 KV 布局或分页问题，而是跨节点、跨角色的 KV 传输路径设计。
- 相比 LMCache，这篇更强调 **分离式 serving 中的带宽再分配**，而不是通用 KV 持久化或跨请求复用。
- 相比 Preble，这篇不是 prefix 调度，而是 agentic、多轮场景下大规模 KV 的传输与装载。
- 和本 repo 的 `Offloading and Disaggregated Serving` 模块最贴近。

## 5. Evidence

- 论文声称在生产式 agentic workload 上，离线推理吞吐最高提升 `1.87x`。
- 在线 serving 吞吐平均提升 `1.96x`，且不违反 SLO。
- 实验覆盖 3 个模型和 production agentic workloads。

## 6. My Take

- 真正新的点在于把 decode 侧“空闲 NIC / 计算网络”变成 KV 传输资源，而不是继续在 prefill 侧挤带宽。
- 这是典型的系统论文: 问题建模、数据路径、调度器三者绑在一起才成立。
- 限制也很明显: 价值高度依赖 disaggregated inference、RDMA 能力和 agentic / multi-turn workload。
- 但只要你的 repo 要覆盖 KV transfer 和 prefill/decode 分离，这篇就应该进主列表。
