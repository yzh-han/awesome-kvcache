## 1. One-Line Summary

这篇论文用真实云厂商 trace 分析 KV cache reuse 模式，并基于 workload 特征设计 eviction policy。

## 2. Why It Matters

它提供了真实生产侧的 reuse、寿命和容量画像，不再只靠 synthetic workload 评估。

## 3. Main Idea

- 分析真实 LLM 服务中的 KV cache 命中与重用分布。
- 观察不同 request category 的 reuse 差异。
- 基于 trace 特征提出 workload-aware eviction policy。

## 4. Relation to Existing Work

- 相比 Marconi / Preble，它更偏 trace characterization。
- 相比 Mooncake / DualPath，它不主打 disaggregation。
- 相比 benchmark 论文，它有真实 workload 证据。

## 5. Evidence

- 公开摘要强调在真实 trace 下提升 serving performance。
