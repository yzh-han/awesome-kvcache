## 1. One-Line Summary

KeyDiff 基于 key similarity 做训练无关的 KV eviction，用 key diversity 代替 attention score 来决定保留哪些 cache。

## 2. Why It Matters

很多 eviction 方法依赖 attention score，但这会限制和高效 attention kernel 的兼容性。  
KeyDiff 的价值在于它完全绕开 attention score，直接用 key 几何结构来做保留决策。

## 3. Main Idea

- 观察到“几何上更独特”的 key 往往更重要。
- 通过 key similarity / diversity 来构造 KV selection 问题。
- 不依赖 attention score，因此和 FlashAttention 一类实现更兼容。
- 目标是在严格 budget 下仍能处理超长 prompt。

## 4. Relation to Existing Work

- 相比 H2O、Lookahead Q-Cache，它不依赖 query-aware importance。
- 相比 CAKE，它不是 layer-aware budget，而是 key-geometry-aware selection。
- 属于 eviction 方向里很有辨识度的一条分支。

## 5. Evidence

- 在 8K cache budget 下，相对 full KV baseline 的 LongBench 性能差距可小于 `0.04%`。
- 同时带来约 `23%` KV cache reduction。
- 相比其他 token-eviction 方法，端到端延迟最多可降低 `30%`。

## 6. My Take

- KeyDiff 很值得收录，因为它给 eviction 提供了一个不依赖 attention score 的新判据。
- 这让它在工程落地上比某些 query-aware 方法更友好。
