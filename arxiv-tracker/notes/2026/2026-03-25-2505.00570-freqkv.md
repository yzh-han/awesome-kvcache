## 1. One-Line Summary

FreqKV 用频域压缩替代 token eviction，迭代压缩不断增长的 KV cache，以支持更稳的超长上下文扩展。

## 2. Why It Matters

很多 KV compression 方法在长上下文外推时会明显退化，尤其 token eviction 容易损失局部关键信息。  
FreqKV 属于这批 `Jan-Feb 2026 accepted` 论文，因为它的 OpenReview 页面在 `2026-01-26` 已明确标为 `ICLR 2026 Poster`，而且它代表的是 repo 里一条很有辨识度的路线：`frequency-domain KV compression`。

## 3. Main Idea

- 观察到上下文信息在频域里主要集中在低频成分。
- 因此不是直接踢掉 token，而是迭代丢弃高频成分来压缩 KV。
- 方法是 `parameter-free` 且 `architecture-agnostic`。
- 配合最少量的 8K length training，可把 LLaMA-2-7B 的上下文扩到 `256K`。
- 同时覆盖 prefilling 和 decoding 两个阶段。

## 4. Relation to Existing Work

- 相比 `H2O` / `SnapKV` 的 token-level importance 路线，FreqKV 不依赖 eviction。
- 相比 `KVSlimmer` 的 merging，FreqKV 更像一种正交变换域上的压缩。
- 相比 `StreamingLLM`，它不是固定窗口和 sink-token 思路，而是长上下文扩展导向的压缩方法。

## 5. Evidence

- OpenReview abstract 声称上下文可扩展到 `256K`
- 在 stable perplexity 下完成 context extension
- 在 prefilling 和 decoding 上都优于现有 KV compression 方法
- 论文强调对 understanding 和 generation 都有效

## 6. My Take

- FreqKV 的优势是路线非常清晰，和主流 token eviction 派别差异大，放在 repo 里辨识度很高。
- 它更像“上下文扩展导向的 KV compression”，比单纯省一点显存更有研究味道。
- 风险在于它对最少量训练的依赖意味着不完全是 drop-in inference patch。
