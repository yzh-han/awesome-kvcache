## 1. One-Line Summary

NEO 把部分 attention compute 和 KV 状态下放到 CPU，并通过 asymmetric GPU-CPU pipelining 和 load-aware scheduling 提升在线 serving 吞吐。

## 2. Why It Matters

它把 `CPU compute + KV offload + online batching` 一起设计，不是只把 CPU 当内存池。

## 3. Main Idea

- 将部分请求的 attention 与 KV 放到 CPU 执行。
- 区分 GPU-request 和 CPU-request 两类路径。
- 用 asymmetric pipelining 减少 GPU/CPU 空泡。
- 用 load-aware scheduling 平衡 CPU 和 GPU 负载。

## 4. Relation to Existing Work

- 相比 SpeCache / ShadowKV，NEO 更偏 online batching 和异构流水线。
- 相比 HGCA，它不做双路 attention 融合。
- 相比 Mooncake / DualPath，它更偏单机异构执行。

## 5. Evidence

- 公开摘要报告在多种 GPU 上有明显 throughput 提升。
- 主要收益是 `throughput↑`、`GPU memory pressure↓`。

## 6. My Take

- NEO 很适合放到 offloading/system 主线。
- 它把“CPU 能不能真正参与在线推理主路径”回答得比较完整。
