## 1. One-Line Summary

RazorAttention 利用少数 retrieval heads 保留全量远程 KV，其余 heads 只保留局部上下文，再配合 compensation token 回补被裁掉的信息，实现训练免调的 KV cache 压缩。

## 2. Why It Matters

很多 KV compression 方法的核心是直接丢 token，这会不可逆地删除未来查询可能需要的信息。  
RazorAttention 的价值在于，它把“哪些 head 真的需要全局信息”显式建模出来。

## 3. Main Idea

- 观察到大多数 attention heads 主要看局部上下文，只有少数 retrieval heads 会检索全局 token。
- 因此采用 head-wise 差异化缓存策略：retrieval heads 保留完整 KV，非 retrieval heads 只保留局部窗口。
- 引入 compensation token 回补非 retrieval heads 丢掉的远程信息。
- 方法是 training-free，并兼容 FlashAttention。

## 4. Relation to Existing Work

- 相比 H2O、CAKE 这类 token-level 方法，它把重点放在 head-level roles。
- 相比 SnapKV，它更强调 attention head 的功能分化，而不是只估计 token 重要性。
- 相比 Palu、MatryoshkaKV，它不是压表示维度，而是改缓存保留策略。

## 5. Evidence

- OpenReview TL;DR 明确写了在一般场景下可达 `3x reduction`。
- 摘要声称 `KV cache size` 可降低 `over 70%`。
- 同时“without noticeable impacts on performance”。

## 6. My Take

- 这篇论文很值得收，因为它提供了一个很有辨识度的视角：不是所有 head 都值得同等成本地维护远程 KV。
- 它适合作为 retrieval-head-based compression 的代表工作。
