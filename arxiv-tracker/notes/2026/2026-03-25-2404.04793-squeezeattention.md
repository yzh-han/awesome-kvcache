## 1. One-Line Summary

SqueezeAttention 把 KV 管理从单一 token 压缩扩展到 “token 维 + layer 维” 两维联合优化，先按层动态分配 budget，再在层内做 token 稀疏化。

## 2. Why It Matters

很多 KV-cache 方法默认所有层拿差不多的预算，但这往往浪费了层间敏感度差异。  
SqueezeAttention 的价值就在于它把问题从“哪些 token 要留”推进到“哪些层更值得拿更多 cache”。

## 3. Main Idea

- 观察到不同 attention layer 对输入 token 的敏感度不同，不该共享同一套 KV budget。
- 用 self-attention 前后 prompt 差异的余弦相似度来估计层重要性。
- 按层重要性把层分组，并动态分配不同的 KV-cache 预算。
- 在每一层内部，再接入已有的 sequence-wise token sparsification 方法进一步压缩。

## 4. Relation to Existing Work

- 相比 H2O、SnapKV，这篇多了一层 `layer-wise budgeting`。
- 相比 CAKE，它更像“先按层分钱，再决定层内怎么省”。
- 相比 Palu、MatryoshkaKV，它优化的是 sequence budget allocation，而不是 hidden dimension。

## 5. Evidence

- 论文声称可实现约 `30% - 70%` 的显存减少。
- 吞吐提升最高可达 `2.2x`。
- 收益来自二维管理，而不是单一 token 稀疏化。

## 6. My Take

- 这篇论文值得收，因为它把 `layer-wise heterogeneity` 变成了显式的 KV 管理信号。
- 它很适合作为 layer-aware budget allocation 子线的代表工作。
