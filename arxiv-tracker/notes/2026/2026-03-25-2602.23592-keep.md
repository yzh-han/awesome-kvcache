## 1. One-Line Summary

这篇论文面向 embodied planning，把长期记忆从 raw text prompt 转成可管理的 KV-cache memory，并通过静态/动态分组、跨组重计算和分层加载来降低 prefill 与 TTFT 开销。

## 2. Why It Matters

它不是通用 LLM serving 论文，而是一个很具体的 memory-augmented planning 场景。但它确实把“频繁更新的 KV cache 怎么组织、何时重算、如何分层加载”当成核心问题，KV cache 不是附带概念。

它也属于这批 `Jan-Feb 2026 accepted` 论文，因为它的 arXiv 版本提交于 `2026-02-27`，并且 arXiv comments 里公开给出了 `DAC 2026` 的接收信号。

## 3. Main Idea

- 指出 embodied planning 常把历史经验以 raw text 方式塞回 prompt，导致 prompt 变长和 prefill 延迟高。
- 提出 `Static-Dynamic Memory Construction`，用 mixed-granularity memory groups 降低 KV 重算成本。
- 提出 `Multi-hop Memory Re-computation`，根据跨 memory group 的重要 cross-attention 逐步重建交互。
- 提出 `Layer-balanced Memory Loading`，缓解不同层上的 KV loading 与 cross-attention 负载不均。
- 整体思路是把 memory 管理、KV 重算和加载平衡一起做，而不是只优化某一环。

## 4. Relation to Existing Work

- 相比 CacheBlend，这篇显式把 KV cache memory management 做成系统核心，并报告更低 TTFT。
- 相比 Preble / LMCache，这篇更偏“任务内长期记忆管理”，而不是跨请求 prefix 复用。
- 相比 PagedAttention，这不是通用 serving 内存布局问题，而是记忆块组织与重算策略。

## 5. Evidence

- 相比 text-based memory 方法，ALFRED 数据集上速度提升 `2.68x`，精度损失可忽略。
- 相比 CacheBlend，据称成功率提升 `4.13%`，TTFT 降低 `1.90x`。

## 6. My Take

- 新意在于它把“KV cache 作为 memory substrate”做成了一整套 embodied planning runtime，而不是简单复用现成 prefix cache。
- 但场景相对专用，外推到通用聊天或通用 serving 还需要谨慎。
- 我倾向于把它留在 tracking，同时在 repo 里保留，因为它对 “task-specific KV memory management” 很有代表性。
