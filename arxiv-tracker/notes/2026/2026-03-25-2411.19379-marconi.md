## 1. One-Line Summary

Marconi 把 prefix caching 扩展到 hybrid LLM，并用 reuse likelihood 与 compute savings 共同驱动 admission / eviction。

## 2. Why It Matters

它把 prefix caching 从“能不能 cache”推进到“怎么更聪明地 cache”。

## 3. Main Idea

- 面向 hybrid attention-recurrent 模型设计 prefix caching。
- admission 和 eviction 同时考虑复用概率与潜在计算收益。
- 目标是提高 token hit rate 并降低 TTFT。

## 4. Relation to Existing Work

- 相比 Preble，它更偏 cache policy 本身。
- 相比 KVCache Cache in the Wild，它偏机制设计，不是 trace characterization。
- 相比 LMCache，它更强调 hybrid LLM 场景。

## 5. Evidence

- MLSys 摘要强调更高 token hit rate 和更低 TTFT。

## 6. My Take

- 这篇值得进入 prefix_reuse_and_prompt_cache 主线。
- 它给 prefix caching 增加了 model-aware policy 视角。
