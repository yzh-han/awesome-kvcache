## 1. One-Line Summary

Lookahead Q-Cache 用低成本 pseudo lookahead query 逼近真实 decode query，从而让 KV eviction 的重要性估计更接近真实推理需求。

## 2. Why It Matters

很多 KV eviction 方法依赖 prefill 阶段的 attention 观察，但真实 decode query 往往并不一致。  
Lookahead Q-Cache 的意义就在于修正这个“prefill-only importance estimation”偏差。

## 3. Main Idea

- 生成低成本 pseudo lookahead queries。
- 用这些伪 query 作为观察窗口来估计 token 重要性。
- 使 eviction 决策更接近真实 decode 阶段的 attention 需求。
- 能和已有方法组合使用，而不是完全替代它们。

## 4. Relation to Existing Work

- 相比 H2O、CAKE，Lookahead Q-Cache 更 query-aware。
- 相比 KeyDiff，它依赖 pseudo query 而不是 key diversity。
- 和后续更一般的 query-aware eviction 路线能很好串起来。

## 5. Evidence

- 在 LongBench 上，在有限 budget 下相对已有方法提升约 `1~4` 分。
- 也在 Needle-in-a-Haystack 上展示了效果。

## 6. My Take

- 这篇论文值得收录，因为它把 eviction 的观察窗口问题说得很清楚。
- 它是 2025 年 query-aware KV eviction 里很有代表性的一篇。
